# Dashboard Feedback UI Changes

## Asset Detail Page changes

### 1. Review controls
Add a version review state control with:
- in review
- approved
- rejected
- needs changes

### 2. Overall carousel feedback section
A comment list and input for whole-carousel feedback.
Use this for:
- flow
- tone
- CTA
- narrative
- strategic notes

### 3. Slide feedback section
When a slide is selected in preview:
- show comments for that slide
- allow adding a comment to that exact slide

### 4. Selected slide state
Carousel preview should track selected slide index.
Selected slide becomes the target for precise feedback.

### 5. Comment status
Allow open/resolved state later. For first pass, creation is most important, resolution can be lightweight.

## UX principle
The dashboard should evolve from static viewer to review workstation.
Feedback should feel native to the asset, not bolted on.
